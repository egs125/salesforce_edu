상단에 떠있는 탭이 일종의 테이블
세일즈포스에서는 테이블로 지칭하지 않고 개체(Object)라고 부름

=>
개체 = 엑셀에서의 시트
레코드 = 하나의 행
필드 = 하나의 컬럼, 열

필드가 모여서 레코드, 레코드가 모여서 개체, 개체가 모여서 하나의 앱을 형성

세일즈포스에서는 필드를 줄인다고 해서 테이블 사이즈가 줄어드는 것이 아니므로 설계 시 테이블 최적화에 크게 신경쓸 필요 없음

레코드 하나의 용량은 무조건 2kb
(필드가 몇개인지 관계 없이 무조건 2kb임)

각 필드마다 data type을 가짐

상단: Compact Layout
우측 하단에 관련 Object data를 표시하는 Related List 

회사 정보 설정 시 사용 통화가 한국어 - null로 설정되어 있으면 빈 껍데기가 화면에 보이는 오류가 있음
사용 통화 활성화 이후에는 비활성화 불가능하므로 구축 시 유의할것

회사 정보 설정 시 기본 시간대 설정하는 것이 굉장히 중요함
=> DB 상에는 GMT 0 기준으로 시간 관련 데이터가 저장됨
   데이터를 다시 불러올 때 설정되어 있는 기본 시간대를 기준으로 변환하여 보여주는 것

신규 유저 생성 시 회사 정보를 기준으로 유저 생성됨


세일즈포스에서의 샌드박스 = 개발환경

"고정" 설정 시 어떤 화면으로 들어갔다가 돌아오더라도 고정 설정된 것을 제일 먼저 보여줌

앱 순서를 바꾸고 싶으면 드래그 해서 배치 변경하면 됨

앱 만들기 (앱 = 엑셀 파일)
=> 앱 관리자 화면으로 이동
   앱 생성 시 개발자 이름이 일종의 API 명이므로 작명 시 유의할것
   
  콘솔 탐색은 탭 형식으로 앱을 띄움
  표준 탐색은 한 화면에 앱을 띄움

  세일즈포스는 기본으로 네이티브 앱을 제공함
  특정 앱은 특정 프로필에만 공유하도록 설정도 가능
  초기 앱 생성 시 설정한 탐색 유형은 생성 이후 변경 불가
  앱에 항목 추가 시 즉각 반영되지는 않음

  설정 메뉴에서 개체 편집 클릭 시 바로 개체 편집 화면으로 이동 가능
  필드명 역시 API명과 연관되므로 필드명 작명 시 유의해야함

  스탠다드 필드는 이름에 영어만 들어가고
  커스텀 필드는 뒤에 __c가 붙음
  * 스탠다드 필드명은 변경이 불가능

  위도, 경도값 저장도 지원. 구글 지도를 기본적으로 지원함

  통화와 숫자 필드의 차이
  => 통화 필드는 앞에 통화 기호를 보여줌

  리치 텍스트 타입으로 지정 시 이미지 삽입도 가능하나 이미지 크기 조정은 불가능
  리치 텍스트는 html 형태로 저장하게 됨

  사용자 정의 개체 설정 시 "레코드 이름" 설정 = 레코드의 키값을 설정하는 행위라고 이해하면 됨
  검색 허용에 체크하면 글로벌 서치(전역 검색) 가능해짐 = 계정에 권한이 있을 시 데이터 검색 가능
  이 사용자 정의 개체 저장 후 새 사용자 정의 탭 마법사 시작에 체크 후 저장하면 자동으로 설정탭으로 넘어감
  탭 설정에서 사용자들에게 해당 개체를 보여줄지 말지 설정함

  사용자 정의 개체 설정 = 엑셀 파일에 시트를 하나 추가함

  축소 레이아웃 할당을 통해 보여줄 필드 편집

  조직 기본값으로 하면 모든 사용자에게 해당 템플릿으로 보여줌

  페이지 레이아웃 설정 화면에서 필터 설정 시 조건별로 화면 표시 여부를 설정 가능



각 개체별로 연결할 수 있는 방법 중 가장 보편적인 것
- lookup relation: 참조 관계로 나름의 부모 관계가 있으나 마스터 릴레이션보다는 느슨함. 상위 개체가 지워져도 자식은 지워지지 않음
- master relation: 상위 개체가 삭제되면 하위 레코드가 모두 자동 삭제

룩업 릴레이션의 장점
- 필드 설정에서 Roll-up 설정 시 별도 계산 없이도 자동으로 합계 계산 가능 등
- 자기 자신을 참조하는 것도 가능
- 기준값으로 잡는 데이터가 자주 변경이 된다면 픽리스트를 쓰는 것보다는 릴레이션 설정을 하는 것이 좋음

스키마 빌더에서 ERD 조회 가능

브라우저 확장 프로그램 중에 Salesforce DevTools 설치 시 해당 프로그램에서도 ERD 조회 가능

필드 및 관계 설정 화면에서 조회 관계 설정 가능
하위 관계 이름 역시 표준에 맞춰서 작명에 유의해야

숫자 타입의 필드 설정 시 길이 및 소수점을 1로 설정하면 아래 자리는 반올림하여 화면에 보여줌. DB 상에는 소수점 그대로 저장하게 됨
  길이 + 소수점을 합쳐 18을 넘어가면 오류 발생함

롤업 설정은 숫자 필드 뿐만 아니라 날짜 필드에서도 설정 가능

개체 관리자 > 페이지 레이아웃에서 어떤 항목을 보여줄지 설정 가능
페이지 편집에서도 어떤 항목을 보여줄지 설정할 수는 있음. 페이지 레이아웃에서는 드래그 앤 드랍 형식으로 필드 추가
=> 페이지 레이아웃 설정 화면에서 필드 추가 시 필드별 가시성 설정 가능

추가 가능한 필드 개수는 800여개 정도 되지만 화면에 한번에 표시할 수 없으므로 한 필드에 여러 값을 넣는 식으로 제한

레코드 유형 = 꼬리표를 달아주는 작업
레코드 유형별로도 별도의 레이아웃 설정 가능

강조 패널

필드 집합 = 필드셋
  특정 필드들만 묶어서 해당 데이터만 업데이트

검색 레이아웃 = 검색 시 나오는 항목들을 설정할 수 있음. 표시 가능 필드 갯수는 10개로 제한됨

버튼, 링크 및 작업 메뉴에서 '새작업' 이란 액션을 정의하는 기능
새 버튼 또는 링크 버튼을 누르면 버튼 클릭 시 다른 페이지로 넘어가는 기능 정의



베이스를 영어로 만들고 번역을 붙이는 식으로 구축. 커스텀 설정과 연관되어 있음
설정 > 탭 및 레이블 이름 변경에서 언어별 이름 설정 가능
표준 탭은 세일즈포스에서 기본으로 제공하는 프리셋이 있음

번역에서 선택 목록 값을 선택하면 픽리스트의 옵션 역시 번역 추가 가능
번역 설정값을 엑셀로 정리해서 일괄로 설정하는 것도 가능

숫자, 요일 등 국가별로 다르게 사용하는 경우에도 숫자 필드 및 통화 필드에 대한 locale 설정 지원

확인 규칙에서 참으로 떨어지면 에러 메시지를 보여주도록 설정 가능
